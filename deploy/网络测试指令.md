# 树莓派网络测试指令（精简版）

## 1) 直接粘贴执行（测试 API 下单通路）

```bash
cd /opt/pm-shadow
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
export no_proxy=localhost,127.0.0.1
export NO_PROXY=localhost,127.0.0.1
curl -4 https://ifconfig.me
curl -6 https://ifconfig.me || true
./.venv/bin/python check_order_connectivity.py --live --usdc 6 --cancel-after-sec 5 --proxy http://127.0.0.1:7890
```

## 2) 根据结果怎么做

- 出现 `RESULT: success (api path works)`  
  - 网络与下单链路正常，可直接启动主策略：
  ```bash
  cd /opt/pm-shadow
  ./.venv/bin/python run_shadow.py --dual --sync-live-cash --reset-state --days 7 --state-db shadow_state.db --out-dir reports --proxy http://127.0.0.1:7890
  ```

- 出现 `RESULT: failed (GEO_BLOCK_403)`  
  - 说明当前进程出口仍被地区风控拦截。先检查 Clash 是否 TUN+全局、再排查 IPv6 泄漏（可临时禁用 IPv6 后重测）。

- 出现 `POLY_API_400 ... lower than the minimum`  
  - 不是地区问题，是下单量不足；把 `--usdc` 调大后重测（建议 `>= 6`）。
